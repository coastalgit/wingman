## Decisions Made

| Decision | Detail |
|----------|--------|
| **Migrate away from Flutter** | Desktop app is too heavy; browser UI is cross-platform, cross-editor, lighter |
| **TUI rejected** | Textual demo was tried and rejected — not visual enough |
| **Web UI chosen** | Node.js/Express serving localhost, browser as display layer |
| **AstroVista theme** | Coral primary `#df6035`, dark base `#1a1a1a`, Outfit/Merriweather/Fira Code fonts |
| **AI-tool agnostic** | Not tied to Claude Code — supports Cursor CLI, Gemini CLI, Aider, etc. |
| **CLI installable** | Primary: `npm install -g wingman`, run from project directory |
| **Per-project data** | All history/config in `.wingman/` within the project directory |
| **Slash commands for integration** | `wingman init` installs `/ccp`, `/ccc` as native commands per AI tool |
| **No TUI, no VS Code extension** | Browser UI serves both terminal and IDE users |
| **Public repo** | https://github.com/coastalgit/wingman, GPL license |
| **Feature branch workflow** | `feature/wingman-v2` as working branch, worktrees for experiments |
| **Obsolete features removed** | Project setup screen, WSL path conversion, speech-to-text, shell aliases, tool selector tabs in header |
| **Core value preserved** | Context management, prompt history, session organization, multi-tool support |

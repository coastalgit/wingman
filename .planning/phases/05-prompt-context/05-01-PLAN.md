# Plan 05-01: Backend — Per-Session Files & REST API

## Goal

Add per-session file storage (cprompt.md, ccontext.md, history.json) and REST API so the session UI can read/write prompt and context data, and inject `/ccp`/`/ccc` into the PTY.

## Requirements Covered

FILE-01, FILE-02, FILE-03, API-01..05

## File Layout

```
.ai/wingman/sessions/<session-id>/
  cprompt.md      # Current staged prompt (overwritten each save)
  ccontext.md     # Current context (overwritten each save)
  history.json    # Array of { id, text, timestamp, tokens }
```

## Tasks

### Task 1: Session directory & file init

In `SessionManager.spawnSession()`, after creating the session:
- Create `.ai/wingman/sessions/<id>/` directory
- Write empty `cprompt.md`, `ccontext.md`, and `history.json` (`[]`)
- Add `getSessionDir(sessionId)` helper method

### Task 2: File read/write methods on SessionManager

- `savePrompt(sessionId, text)` — write cprompt.md
- `loadPrompt(sessionId)` — read cprompt.md, return `''` if missing
- `saveContext(sessionId, text)` — write ccontext.md
- `loadContext(sessionId)` — read ccontext.md, return `''` if missing
- `appendHistory(sessionId, entry)` — read history.json, push entry, write back
- `loadHistory(sessionId)` — read history.json, return `[]` if missing

### Task 3: REST API routes in server.js

| Method | Path | Action |
|--------|------|--------|
| GET | `/api/sessions/:id/prompt` | Return cprompt.md content as `{ text }` |
| POST | `/api/sessions/:id/prompt` | Save cprompt.md, append to history.json, write `/ccp\n` to PTY stdin |
| GET | `/api/sessions/:id/context` | Return ccontext.md content as `{ text }` |
| POST | `/api/sessions/:id/context` | Save ccontext.md, write `/ccc\n` to PTY stdin |
| GET | `/api/sessions/:id/history` | Return history.json array |

POST prompt body: `{ text }` — server generates id (timestamp), estimates tokens (~text.length/4), appends to history, then injects `/ccp\n` into PTY.

POST context body: `{ text }` — server saves and injects `/ccc\n` into PTY.

Validate session exists (404 if not). Validate PTY is alive before inject (400 if stopped).

### Task 4: express.json() middleware

Ensure `app.use(express.json())` is present in server.js (may already be there).

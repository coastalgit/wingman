# Plan 05-02: Frontend — Session UI with Embedded Terminal

## Goal

Replace the current bare terminal session page with the original Wingman web UI layout (from commit 84f042b). Two tabs (Session Context / Prompts), no session sidebar, terminal in the bottom third.

## Requirements Covered

SP-01..05, CTX-01..03, CTX-07, PMT-01, PMT-02, PMT-06, PMT-07, PMT-08, PMT-09

## Reference

The original UI (commit 84f042b) has this layout:

```
Header: [W Wingman] [Session Context | Prompts] [session-name] [menu]
+--sidebar--+--main-content------------------------------------+
| Sessions   | Context: [Editor textarea | Markdown preview]   |
|            | Prompts: [History list | Compose textarea]       |
+------------+-------------------------------------------------+
Status bar
```

The new session page keeps the same layout but:
- **Remove sidebar** (sessions managed via Mission Control)
- **Add terminal** in bottom third of main content
- Main content becomes a vertical split: editor area (top ~67%) + terminal (bottom ~33%)

New layout:

```
Header: [W Wingman] [Session Context | Prompts] [session-desc]
+--main-content------------------------------------------------+
| Context: [Editor | Preview]   (or Prompts: [History | Edit]) |
|                                                              |
+--------------------------------------------------------------+
| Terminal (xterm.js)                                          |
|                                                              |
+--------------------------------------------------------------+
Status bar
```

## Tasks

### Task 1: New session.html

Rebuild `public/session.html` based on the original `index.html` from 84f042b:
- Header with brand, feature-nav tabs (Session Context / Prompts), session description
- Main content area split vertically: panel area (top) + terminal div (bottom)
- Context panel: split pane with editor textarea + markdown preview
- Prompts panel: split pane with history list + compose textarea
- Status bar at bottom
- Include xterm.js CDN links, marked.js CDN link
- Include `terminal.js`, and new `session-ui.js`

### Task 2: Styles

Port the relevant styles from the original `styles.css` (84f042b) into the session page:
- Feature nav tabs
- Panel split layout (editor + preview, history + compose)
- Textarea styling (code-textarea class)
- Markdown preview styling
- History list styling
- Toolbar styling (char count, token estimate)
- Button styling (save-btn, clear-btn)
- Adapt for vertical split: top panel area + bottom terminal

Add new styles for the terminal section:
- `.terminal-section` — fixed bottom third, border-top separator
- Terminal fills its section fully (as current session.html does)

### Task 3: session-ui.js — Tab switching & editor setup

New file `public/session-ui.js`:
- Feature nav tab switching (Context / Prompts) — show/hide panels
- Context editor: live markdown preview via `marked.parse()` on input
- Context editor: char count + token estimate (~length/4) in toolbar
- Prompt editor: char count + token estimate in toolbar
- Clear button: reset prompt textarea
- History search: filter history list on input

### Task 4: Adapt terminal.js

- Terminal now lives inside `#terminal` div within the new layout (not full page)
- FitAddon must fit to the terminal section, not the full viewport
- On window resize, re-fit terminal
- Export sessionId so session-ui.js can use it for API calls

### Task 5: History list rendering

In session-ui.js:
- Fetch history from `GET /api/sessions/:id/history`
- Render as scrollable list (newest first): first-line preview, timestamp, token count
- Click handler: load full text into compose textarea, switch to compose
- Search input filters by text content

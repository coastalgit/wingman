# Plan 05-03: Integration — Wire Save to API, Inject into PTY, End-to-End

## Goal

Wire the Save buttons to the backend API, verify `/ccp` and `/ccc` auto-inject into the terminal, and test the full flow end-to-end.

## Requirements Covered

CTX-04, CTX-05, CTX-06, PMT-03, PMT-04, PMT-05, PMT-10

## Tasks

### Task 1: Wire "Save prompt" button

In session-ui.js:
1. On click (or Ctrl+Enter): read compose textarea
2. `POST /api/sessions/:id/prompt` with `{ text }`
3. Server saves cprompt.md, appends to history.json, writes `/ccp\n` to PTY
4. On success: clear textarea, refresh history list, show toast "Prompt saved"

### Task 2: Wire "Save context" button

In session-ui.js:
1. On click: read context textarea
2. `POST /api/sessions/:id/context` with `{ text }`
3. Server saves ccontext.md, writes `/ccc\n` to PTY
4. On success: show toast "Context saved"

### Task 3: Load data on page open

In session-ui.js `init()`:
1. `GET /api/sessions/:id/context` → populate context textarea + preview
2. `GET /api/sessions/:id/history` → render history list
3. `GET /api/sessions/:id/prompt` → populate compose textarea (if user had unsent draft)

### Task 4: Keyboard shortcut

- Ctrl+Enter in compose textarea triggers Save prompt
- Prevent default form submission

### Task 5: Copy to testbed & manual UAT

1. Copy changed source files to testbed
2. Start server, create session from Mission Control
3. Verify: session page shows context tab with editor + preview
4. Verify: switch to prompts tab, write prompt, Save → `/ccp` appears in terminal, Claude reads it
5. Verify: switch to context tab, write context, Save → `/ccc` appears in terminal
6. Verify: history list shows saved prompts, clicking one loads it
7. Verify: close tab, reopen → context and history load from disk
8. Verify: stop/restart server → data persists

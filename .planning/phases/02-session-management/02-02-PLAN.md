# Phase 2 Research (02-RESEARCH.md)

WebSocket Reconnection Handshake (Pattern 2):
- Browser sends: { type: "handshake", sessionId: null | uuid }
- Server responds: { type: "handshake-ack", sessionId: uuid, status: "new|resumed|closed", history: [...], createdAt: ISO8601 }
- Browser stores sessionId in localStorage on handshake-ack
- On page reload, browser retrieves sessionId and sends in next handshake
- History buffer contains raw PTY output as strings; replay via term.write(line) for each element

Key implementation details from research:
- localStorage.getItem('wingmanSessionId') returns null on first connect; returns UUID on reconnect
- handshake message sent in ws.onopen after creating Terminal
- handshake-ack received before output messages (exclusive first message on reconnect)
- history array replay with terminal.writeln('[Replaying history...]') followed by forEach write
- Session metadata displayed in document.title and optional #session-info div

Extend ws.onmessage to handle handshake-ack BEFORE other messages:
```javascript
ws.onmessage = (event) => {
  const msg = JSON.parse(event.data);

  // Handle handshake-ack first (should be the first message on any connect/reconnect)
  if (msg.type === 'handshake-ack') {
    // Persist session ID in localStorage for future reconnects
    localStorage.setItem('wingmanSessionId', msg.sessionId);

    // Display session info (title and optional status div)
    const shortId = msg.sessionId.substring(0, 8);
    document.title = `Wingman - Session ${shortId}...`;

    // If session-info div exists in HTML, update it
    const statusDiv = document.getElementById('session-info');
    if (statusDiv) {
      const created = new Date(msg.createdAt).toLocaleString();
      statusDiv.textContent = `Session: ${msg.sessionId} | Created: ${created}`;
    }

    // Replay history if reconnecting (status: "resumed" or if history array is non-empty)
    if (msg.history && msg.history.length > 0) {
      term.writeln('[Replaying session history...]');
      msg.history.forEach(line => term.write(line));
      term.writeln('[End of history]');
    }

    console.log(`Connected to session ${msg.sessionId} (status: ${msg.status})`);
    return;  // Don't process other message types in handshake-ack
  }

  // Handle output (existing behavior, unchanged)
  if (msg.type === 'output') {
    term.write(msg.data);
  }

  // Handle session-ended
  if (msg.type === 'session-ended') {
    term.writeln('\r\n[Session ended]');
  }
};
```

Key details:
- localStorage.getItem('wingmanSessionId') returns null if key not set (first connect)
- localStorage.setItem() persists key across page reload and tab close
- JSON handshake message is sent BEFORE resize and input messages
- handshake-ack is checked first in onmessage to avoid treating it as output
- History replay includes terminal.writeln() markers ('[Replaying...]' / '[End of history]') for user feedback
- document.title updated to show session ID (visible in browser tab)
- #session-info div update is conditional (graceful degradation if HTML doesn't have this div)

Warning: Do not send 'input' messages during history replay (avoid double-echoing) — history contains the original output, not user input.

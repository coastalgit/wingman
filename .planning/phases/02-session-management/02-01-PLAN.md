SessionManager pattern:
- Singleton class managing Map<sessionId, session> with in-memory tracking
- Methods: generateSessionId(), spawnSession(ptyProcess), getSession(sessionId), addToHistory(sessionId, data), getHistory(sessionId), closeSession(sessionId)

- Use crypto.randomUUID() (built-in Node 14.17.0+) for session IDs
- .ai/wingman directory created recursively if missing
- sessions.json contains only metadata (id, createdAt, closed) — ptyProcess is in-memory only
- Atomic file writes: write to .tmp file, renameSync to final path

Class structure:
- constructor(projectRoot = process.cwd()): Initialize .ai/wingman directory, create empty sessions.json if missing, load existing sessions from disk via loadSessionsFromDisk()
- generateSessionId(): return crypto.randomUUID() — no parameters
- spawnSession(ptyProcess, metadata = {}): Generate new UUID, create session object with id/createdAt/ptyProcess/history[]/closed:false, store in this.sessions Map, call updateSessionsFile(), return sessionId
- getSession(sessionId): return this.sessions.get(sessionId) — returns session object or undefined
- addToHistory(sessionId, data): Lookup session, append data to history[], trim to 10,000 max with shift() if overflow
- getHistory(sessionId): Lookup session, return history[] (or empty [] if session not found)
- closeSession(sessionId): Mark session.closed = true, call updateSessionsFile()
- updateSessionsFile(): Extract metadata from all sessions: [{ id, createdAt, closed }], write to .tmp file, atomically rename to .ai/wingman/sessions.json
- loadSessionsFromDisk(): Read sessions.json if exists, parse JSON, restore metadata (ptyProcess set to null, history to []), catch and log errors on parse failure

Key details:
- Use const crypto = require('crypto') for randomUUID()
- Use const fs = require('fs'), const path = require('path') for file I/O
- this.historyLimit = 10000 as instance variable
- this.sessions = new Map() for in-memory tracking
- All methods are synchronous (sync file I/O) — Phase 2 PoC
- No dependencies beyond Node.js built-ins

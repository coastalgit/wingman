**lib/manual-mode.js:**
Create module per Research pattern:

```javascript
function initManualMode(wingmanDir) {
  const promptPath = path.join(wingmanDir, 'cprompt.md');
  const contextPath = path.join(wingmanDir, 'ccontext.md');

  if (!fs.existsSync(promptPath)) {
    fs.writeFileSync(promptPath, '# Prompt\n\nWrite your prompt here.\n');
  }
  if (!fs.existsSync(contextPath)) {
    fs.writeFileSync(contextPath, '# Context\n\nWrite your context here.\n');
  }

  return { promptPath, contextPath };
}
```

Export `initManualMode`.

**server.js changes:**

1. Parse `--manual` flag: `const MANUAL_MODE = process.argv.includes('--manual');` near top of file.
2. After sessionManager initialization, if `MANUAL_MODE`:
   - Call `initManualMode(sessionManager.sessionsDir)` (uses `.ai/wingman/` directory).
   - Log: "Wingman started in manual mode. Session files:" then log the paths.
3. Add `GET /api/mode` endpoint that returns `{ manual: MANUAL_MODE }`. This lets the Mission Control client know the mode.
4. In the `POST /api/sessions` endpoint, if `MANUAL_MODE`: do NOT spawn PTY. Instead return `{ error: 'Manual mode active — no Claude sessions spawned', manual: true }` with status 400.
5. Console output on startup should indicate mode: "Wingman (manual mode) running at..." vs "Wingman running at...".

**public/mission-control.html changes:**
Add a `<div id="mode-banner" class="mode-banner" style="display:none;"></div>` in the header area, between the title and controls. This will show "Manual Mode" indicator when active.

**public/mission-control.js changes:**
1. On page load (or after WebSocket connects), fetch `GET /api/mode`.
2. If `response.manual === true`:
   - Show the mode banner: set `display: block`, textContent = "MANUAL MODE — No Claude sessions spawned. Files at .ai/wingman/cprompt.md and .ai/wingman/ccontext.md"
   - Disable or hide the "New Session" button (since PTY sessions cannot be created).
   - Adjust empty state message: "Manual mode active. Edit prompt/context files in .ai/wingman/ and use /ccc, /ccp in your Claude Code session."
3. If `response.manual === false`: keep default behavior (button enabled, normal empty state).

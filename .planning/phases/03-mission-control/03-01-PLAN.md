<objective>
Build the two-page Mission Control architecture: dashboard at `/` and terminal sessions at `/session/:id`. This is the core UI and server routing that all other Phase 3 plans depend on.

Purpose: Mission Control is the launcher and test harness for all session management. It replaces the current "open terminal on connect" pattern with an explicit "create session from dashboard" pattern.
Output: Working Mission Control dashboard that can list, create, open, close sessions, and shut down Wingman.

Note on MC-01: MC-01 ("npx wingman opens Mission Control") is satisfied here by `node server.js` opening Mission Control in the browser. The `npx` installability wrapper is a Phase 4 (Distribution) concern — this plan implements the functional behavior (server start -> browser opens Mission Control).

**lib/session-manager.js changes:**
Add two new methods to SessionManager class:
1. `getSessionStatus(sessionId)` — derives status from in-memory state:
   - `!session.closed && session.ptyProcess` = `'active'`
   - `session.closed && session.history.length > 0` = `'reconnectable'`
   - else = `'closed'`
2. `getAllSessionsWithStatus()` — returns array of `{ id, description, createdAt, status }` for all sessions. Description defaults to `'Session ' + id.substring(0, 8)` if not set.

**server.js changes:**
1. Add `express.json()` middleware (needed for POST body parsing).
2. Move `express.static` AFTER custom routes to avoid index.html interception (per Research pitfall 5).
3. Add route: `GET /` serves `public/mission-control.html`.
4. Add route: `GET /session/:id` serves `public/session.html`.
5. Add REST API endpoints:
   - `GET /api/sessions` — returns `sessionManager.getAllSessionsWithStatus()`.
   - `POST /api/sessions` — accepts `{ description }` body, spawns PTY (`pty.spawn(BASH_PATH, ['-c', 'claude'], {...})`), calls `sessionManager.spawnSession(ptyProcess, { description })`, registers `ptyProcess.onExit` handler (same pattern as current WS handshake), broadcasts session update to MC clients, returns `{ sessionId, url: '/session/' + sessionId }`.
   - `DELETE /api/sessions/:id` — kills PTY if active, calls `sessionManager.closeSession(id)`, broadcasts session update to MC clients.
   - `POST /api/shutdown` — responds `{ status: 'shutting-down' }`, then calls `gracefulShutdown()`.
6. Extract the existing SIGINT shutdown logic into a `gracefulShutdown()` function. Before killing PTYs, broadcast `{ type: 'shutdown' }` to all WebSocket clients. After killing PTYs and closing WS, close server.
7. Add WebSocket message routing for Mission Control clients:
   - On `{ type: 'mc-connect' }`: set `ws.clientType = 'mc'`. Send back `{ type: 'session-update', sessions: sessionManager.getAllSessionsWithStatus() }`.
   - Existing handshake logic continues to work for session terminals (where `msg.type === 'handshake'`).
8. Add `broadcastSessionUpdate()` helper: iterates `wss.clients`, sends `{ type: 'session-update', sessions }` to clients where `client.clientType === 'mc'`.
9. CRITICAL: Remove the "spawn PTY on null sessionId" block from the WebSocket handshake handler. New sessions are ONLY created via `POST /api/sessions`. The WS handshake handler should ONLY handle reconnection to existing sessions (when sessionId matches a known session). If sessionId is null or unknown, send back an error: `{ type: 'error', message: 'Unknown session. Launch from Mission Control.' }`.
10. Update `import('open')` call to open `http://localhost:${PORT}` (this now opens Mission Control, not a terminal).

**public/session.html:** Rename `public/index.html` to `public/session.html`. Content stays the same.

**public/terminal.js changes:**
Update session ID source to read from URL path first, then fall back to localStorage:
```javascript

const urlSessionId = pathMatch ? pathMatch[1] : null;
const storedSessionId = urlSessionId || localStorage.getItem('wingmanSessionId') || null;
```
Also add handling for `{ type: 'shutdown' }` message:
```javascript
if (msg.type === 'shutdown') {
  term.writeln('\r\n\x1b[31m[Wingman shutting down...]\x1b[0m');
}
```
And `{ type: 'error' }`:
```javascript
if (msg.type === 'error') {
  term.writeln('\r\n\x1b[31m[Error: ' + msg.message + ']\x1b[0m');
}
```

   - Closed sessions: display only, no action buttons.

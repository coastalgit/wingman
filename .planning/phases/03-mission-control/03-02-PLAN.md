  <action>
**lib/process-lock.js:**
Create a new module with two exported functions per the Research pattern:

1. `acquireLock(lockPath, port)`:
   - If lock file exists at `lockPath`, read and parse JSON (`{ pid, port }`).
   - Check if process is alive: `process.kill(existingPid, 0)`. This throws if process is dead.
   - If alive: `console.log('Wingman already running at http://localhost:' + existingPort)` and `process.exit(0)`. (PROC-02: prints existing URL)
   - If dead (catch block): stale lock. Delete the file with `fs.unlinkSync(lockPath)`. Log: "Cleaned up stale lock file". (PROC-03: stale cleanup)
   - After stale cleanup or if no lock exists: write `JSON.stringify({ pid: process.pid, port })` to `lockPath`. (PROC-01: single instance)
   - Ensure parent directory exists (`fs.mkdirSync(path.dirname(lockPath), { recursive: true })`).

2. `releaseLock(lockPath)`:
   - Try `fs.unlinkSync(lockPath)`. Catch and ignore errors (file may already be gone).

**server.js changes:**

1. At top of file, require process-lock: `const { acquireLock, releaseLock } = require('./lib/process-lock.js');`
2. Define lock path: `const lockPath = path.join(process.cwd(), '.ai', 'wingman', 'wingman.pid');`
3. BEFORE `server.listen()`, call `acquireLock(lockPath, PORT)`. This blocks duplicate launches.
4. Enhance the `gracefulShutdown()` function (created in Plan 01):
   - Step 1: Broadcast `{ type: 'shutdown' }` to ALL WebSocket clients (both MC and session types). (PROC-05)
   - Step 2: Kill ALL active PTY processes via `sessionManager.sessions.forEach`. For each session with an active ptyProcess, call `ptyProcess.kill()` then `sessionManager.closeSession(id)`. (PROC-06: zombie prevention)
   - Step 3: Terminate all WebSocket connections.
   - Step 4: Call `releaseLock(lockPath)`. (PROC-04: lock cleanup)
   - Step 5: Close HTTP server, force exit after 500ms timeout.
5. Wire `gracefulShutdown()` to both `process.on('SIGINT')` and `process.on('exit')`:
   - SIGINT: call gracefulShutdown() (primary path for Ctrl+C). (PROC-04)
   - `process.on('exit')`: call `releaseLock(lockPath)` as a safety net (exit handler cannot do async work, but unlinkSync is sync). Do NOT call full gracefulShutdown from exit handler (it's sync-only).
6. Add `process.on('uncaughtException')` handler: log the error, call gracefulShutdown(). Prevents PID file from persisting on crashes.

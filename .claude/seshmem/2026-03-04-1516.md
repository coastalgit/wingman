### Architecture Decided
- **Stack**: node-pty + xterm.js + ws + Express 4 (same as VS Code terminal)
- **PTY mandatory**: Claude Code detects TTY — plain `child_process.spawn` breaks slash commands, spinners, colors
- **Windows/WSL bridge**: Spawn `wsl.exe -e claude` via node-pty on Windows side
- **Session reconnection**: Server-side ring buffer replayed on WebSocket reconnect
- **Mission Control**: Separate page, each session opens in its own browser window
- **PID lock file**: `.ai/wingman/wingman.pid` prevents duplicate instances
- **Session registry**: `.ai/wingman/sessions.json` — all instances read/write, PID-checked on startup

### File Structure (agreed)
```
.ai/wingman/
├── wingman.pid          ← single instance lock
├── sessions.json        ← session registry
└── sessions/
    └── <session-id>/
        ├── cprompt.md   ← manual mode: ccp reads this
        └── ccontext.md  ← manual mode: ccc reads this
```
